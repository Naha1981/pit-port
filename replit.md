# Pit-to-Port Weighbridge Reconciler

An industrial logistics compliance tool for South African mining operations that automatically detects weight discrepancies and revenue leakage between mine departure and port arrival using Gemini AI document extraction.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm --filter @workspace/pit-to-port run dev` — run the frontend (port 18533)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string (auto-provisioned by Replit)
- Required env: `GEMINI_API_KEY` — Google Gemini API key for document extraction

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5 + Multer (multipart file uploads)
- DB: PostgreSQL + Drizzle ORM
- AI: Google Gemini 2.5 Flash via direct REST API (vision/document extraction)
- Frontend: React + Vite + TailwindCSS + shadcn/ui + Recharts
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)

## Where things live

- `lib/api-spec/openapi.yaml` — OpenAPI contract (source of truth)
- `lib/db/src/schema/reconciliation-logs.ts` — DB schema for reconciliation records
- `artifacts/api-server/src/routes/reconciliation.ts` — All reconciliation API routes
- `artifacts/api-server/src/lib/gemini.ts` — Gemini API client for document extraction
- `artifacts/api-server/src/lib/reconciliation-engine.ts` — Variance calculation & status logic
- `artifacts/pit-to-port/src/` — React frontend

## Architecture decisions

- **Gemini via direct REST**: Document parsing delegates to Gemini 2.5 Flash for visual OCR. Both mine slip and port slip are extracted concurrently via `Promise.all`.
- **Server-side reconciliation**: All variance calculation, truck reg normalization, and status assignment happens in `reconciliation-engine.ts` — deterministic, testable, no AI involved.
- **Multer memory storage**: Files are held in memory buffers and forwarded directly to Gemini as base64 — no disk I/O needed.
- **lib/api-zod DOM types**: `lib/api-zod/tsconfig.json` includes `dom` in the lib to support `File`/`Blob` types generated by Orval for the multipart upload schema.
- **Snake_case API, camelCase DB**: The Drizzle schema uses camelCase (JS convention); the `toApiLog()` mapper in the route file converts to snake_case for the OpenAPI contract.

## Product

- Upload mine weighbridge slip + port siding receipt (PDF or image)
- Gemini extracts truck reg, consignment note, weights, and timestamps from both documents
- Reconciliation engine calculates variance % and assigns status: RECONCILED / WARNING / CRITICAL
- Full audit log with search by truck reg, status filter, variance trend chart
- Human-in-the-loop correction: operators can override extracted values and recalculate status
- Delete individual logs from the audit trail

## Gotchas

- Run `pnpm --filter @workspace/api-spec run codegen` after any OpenAPI spec change, then rebuild libs with `pnpm run typecheck:libs`
- `lib/api-zod/tsconfig.json` must include `"dom"` in lib array — Orval generates `File`/`Blob` types for multipart schemas
- The `/reconcile` endpoint uses `multer` — do NOT send JSON, always use `FormData`
- Stats endpoint (`/reconciliations/stats`) must be registered BEFORE `/:id` to avoid route shadowing
